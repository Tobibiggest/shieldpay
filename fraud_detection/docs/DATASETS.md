# Datasets

## Synthetic relational (default, ships with this repo)

Generated by `fraud_detection.data.generators.relational_synthetic_generator`. Unlike
the original `AI_model_Py_Scripts/fraud_dataset_Generator_using_numpy.csv` (flat,
no entity IDs, artificially balanced 50/50), this generator produces:

- Entity ID columns (`sender_id`, `recipient_id`, `device_id`, `ip_address`,
  `merchant_id`, `timestamp`) drawn from small, Zipf-skewed pools so IDs repeat
  across transactions — the prerequisite for any graph to exist.
- Explicitly planted fraud rings: small clusters of accounts sharing a device/IP,
  transacting with a common recipient in a tight time window. This is what gives
  GraphSAGE/GAT/HGT structural signal to learn, not just per-transaction features.
- Realistic imbalance (~3% fraud by default) instead of oversampled 50/50.

Generate it via:
```bash
python -m fraud_detection.cli generate-data --output data/synthetic_relational.csv
```

## IEEE-CIS Fraud Detection (optional second benchmark, Phase 7)

Chosen over PaySim (also fully synthetic — adds little beyond our own generator) and
Elliptic (node-level AML labels covering only ~23% of nodes, not transaction-level).
IEEE-CIS is real transaction-level data (~3.5% fraud) with an identity table whose
`card1`/`DeviceInfo`/`addr1`/`P_emaildomain` fields map onto entity-like proxies.

This repo does **not** download it for you — Kaggle competition data requires your
own Kaggle account/API credentials. To use it:

1. Accept the competition rules and download `train_transaction.csv` +
   `train_identity.csv` from
   https://www.kaggle.com/c/ieee-fraud-detection/data (requires your Kaggle login —
   fetch this yourself in a browser or via your own `kaggle.json` API credentials).
2. Place both CSVs under `fraud_detection/data/ieee_cis/`.
3. Use `fraud_detection.data.adapters.ieee_cis.IEEECISAdapter` (added in Phase 7) to
   load and map them into the same canonical schema the synthetic dataset uses — no
   model code changes required.
